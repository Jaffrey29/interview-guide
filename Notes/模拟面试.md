# 模拟面试接口复现式讲解

## 0. 先建立读代码顺序

不要从 Controller 里硬找 Redis/DB 细节。正确顺序：

```text
Controller 看接口入口
  -> Service 看业务怎么实现
  -> Cache/Repository 看数据怎么读写
  -> Entity 看数据库字段
```

Controller 只做三件事：

```java
// 1. 暴露接口地址
@GetMapping("/api/interview/sessions/{sessionId}/question")
public Result<Map<String, Object>> getCurrentQuestion(@PathVariable String sessionId) {
    // 2. 把请求参数交给 Service
    // 3. 把 Service 返回值包装成统一响应
    return Result.success(sessionService.getCurrentQuestionResponse(sessionId));
}
```

真正链路在 `InterviewSessionService`。

## 1. 为什么 sessionService、sessionCache 能直接调用方法

看 `InterviewController`：

```java
@RestController
@RequiredArgsConstructor
public class InterviewController {
    private final InterviewSessionService sessionService;
    private final InterviewHistoryService historyService;
    private final InterviewPersistenceService persistenceService;
}
```

关键是 `@RequiredArgsConstructor`：

```java
// Lombok 会给 final 字段生成构造器，等价于你手写：
public InterviewController(InterviewSessionService sessionService,
                           InterviewHistoryService historyService,
                           InterviewPersistenceService persistenceService) {
    this.sessionService = sessionService;
    this.historyService = historyService;
    this.persistenceService = persistenceService;
}
```

Spring 看到构造器后，会从容器里找对应 Bean 注入进来。

所以 `sessionService.getCurrentQuestionResponse(sessionId)` 能调用，是因为：

1. `sessionService` 的类型是 `InterviewSessionService`。
2. `InterviewSessionService` 被 `@Service` 注册成 Spring Bean。
3. `InterviewSessionService` 里定义了 `getCurrentQuestionResponse` 方法。

同理，看 `InterviewSessionService`：

```java
@Service
@RequiredArgsConstructor
public class InterviewSessionService {
    private final InterviewQuestionService questionService;
    private final AnswerEvaluationService evaluationService;
    private final InterviewPersistenceService persistenceService;
    private final InterviewSessionCache sessionCache;
    private final ObjectMapper objectMapper;
    private final EvaluateStreamProducer evaluateStreamProducer;
    private final LlmProviderRegistry llmProviderRegistry;
}
```

`sessionCache.saveSession(...)` 能调用，是因为：

1. `sessionCache` 字段类型是 `InterviewSessionCache`。
2. `InterviewSessionCache` 类上有 `@Service`。
3. `InterviewSessionCache` 里定义了 `saveSession(...)` 方法。
4. `@RequiredArgsConstructor` 让 Spring 把 `InterviewSessionCache` 注入进来。

## 2. sessionCache.saveSession 为什么传这些参数

先看方法定义，不要只看调用：

```java
public void saveSession(String sessionId,
                        String resumeText,
                        Long resumeId,
                        List<InterviewQuestionDTO> questions,
                        int currentIndex,
                        SessionStatus status) {
```

这 6 个参数刚好组成一个 Redis 会话快照：

```java
public static class CachedSession implements Serializable {
    private String sessionId;      // Redis key 里也有，但对象内部也保留，方便返回 DTO
    private String resumeText;     // 评估报告时需要简历文本
    private Long resumeId;         // 用于 interview:resume:{resumeId} 找未完成会话
    private String questionsJson;  // Redis 里不直接存 List，而是存题目 JSON
    private int currentIndex;      // 当前做到第几题
    private SessionStatus status;  // CREATED / IN_PROGRESS / COMPLETED / EVALUATED
}
```

为什么不传 `score`、`feedback`、`report`？

因为创建会话时还没答题，也没评估，只有：

```text
会话 ID、简历文本、简历 ID、题目列表、当前题下标、会话状态
```

所以调用处是：

```java
sessionCache.saveSession(
    sessionId,              // 刚生成的会话 ID，用来定位一次面试
    request.resumeText() != null ? request.resumeText() : "",
    request.resumeId(),     // 可为空；有简历时用于查未完成会话
    questions,              // LLM 生成出来的题目列表
    0,                      // 新会话从第 0 题开始
    SessionStatus.CREATED   // 新会话初始状态
);
```

`saveSession` 内部：

```java
public void saveSession(String sessionId, String resumeText, Long resumeId,
                        List<InterviewQuestionDTO> questions, int currentIndex,
                        SessionStatus status) {
    // Redis 的主 key：interview:session:{sessionId}
    String key = buildSessionKey(sessionId);

    // 把传进来的 6 个业务值组装成缓存对象
    CachedSession cachedSession = new CachedSession(
        sessionId, resumeText, resumeId, questions, currentIndex, status, objectMapper
    );

    // 写入 Redis，并设置 24 小时过期
    redisService.set(key, cachedSession, SESSION_TTL);

    // 如果有关联简历，而且会话未完成，再建一个 resumeId -> sessionId 的映射
    // 这样 GET /unfinished/{resumeId} 可以快速查到未完成会话
    if (resumeId != null && isUnfinishedStatus(status)) {
        saveResumeSessionMapping(resumeId, sessionId);
    }
}
```

`CachedSession` 构造器里真正把题目列表转成 JSON：

```java
public CachedSession(String sessionId, String resumeText, Long resumeId,
                     List<InterviewQuestionDTO> questions, int currentIndex,
                     SessionStatus status, ObjectMapper objectMapper) {
    this.sessionId = sessionId;
    this.resumeText = resumeText;
    this.resumeId = resumeId;
    this.currentIndex = currentIndex;
    this.status = status;

    // Redis 缓存中保存的是 questionsJson，不是 Java List
    this.questionsJson = objectMapper.writeValueAsString(questions);
}
```

## 3. 创建会话 POST /api/interview/sessions

Controller：

```java
@PostMapping("/api/interview/sessions")
@RateLimit(dimension = RateLimit.Dimension.GLOBAL, count = 5)
@RateLimit(dimension = RateLimit.Dimension.IP, count = 5)
public Result<InterviewSessionDTO> createSession(@RequestBody CreateInterviewRequest request) {
    InterviewSessionDTO session = sessionService.createSession(request);
    return Result.success(session);
}
```

Controller 只把请求体转成 `CreateInterviewRequest`，然后交给 Service。

请求 DTO：

```java
public record CreateInterviewRequest(
    String resumeText,      // 简历文本；无简历面试可以为空
    @Min(3) @Max(20)
    int questionCount,      // 要生成几道题
    Long resumeId,          // 简历表 ID；可为空
    Boolean forceCreate,    // 是否强制新建，跳过未完成会话检查
    String llmProvider,     // 用哪个 LLM
    @NotBlank
    String skillId,         // 面试方向，如 java-backend
    String difficulty,      // junior / mid / senior
    List<CategoryDTO> customCategories,
    String jdText
) {}
```

Service 复现代码：

```java
public InterviewSessionDTO createSession(CreateInterviewRequest request) {
    // 如果传了 resumeId，说明这次面试和某份简历关联
    // 如果 forceCreate 不是 true，就先检查这份简历有没有未完成会话
    if (request.resumeId() != null && !Boolean.TRUE.equals(request.forceCreate())) {
        Optional<InterviewSessionDTO> unfinishedOpt = findUnfinishedSession(request.resumeId());

        // 有未完成会话就直接返回，避免同一份简历重复创建多个进行中的面试
        if (unfinishedOpt.isPresent()) {
            return unfinishedOpt.get();
        }
    }

    // 生成一个短 sessionId
    // UUID 原始值太长且带横线，这里去掉横线后截 16 位
    String sessionId = UUID.randomUUID().toString().replace("-", "").substring(0, 16);

    // skillId 没传就用默认面试方向
    String skillId = request.skillId() != null
        ? request.skillId()
        : InterviewDefaults.SKILL_ID;

    // difficulty 没传就用默认难度
    String difficulty = request.difficulty() != null
        ? request.difficulty()
        : InterviewDefaults.DIFFICULTY;

    // 取历史题，用于生成新题时去重
    // 有 resumeId：查这份简历 + 这个方向的历史题
    // 没 resumeId：查这个方向的通用历史题
    List<HistoricalQuestion> historicalQuestions =
        persistenceService.getHistoricalQuestions(skillId, request.resumeId());

    // 调题目生成服务
    // 这里传入的参数，都是生成题目所需上下文
    List<InterviewQuestionDTO> questions = questionService.generateQuestionsBySkill(
        request.llmProvider(),       // 选择哪个 LLM
        skillId,                     // 面试方向
        difficulty,                  // 难度
        request.resumeText(),        // 简历文本
        request.questionCount(),     // 题目数量
        historicalQuestions,         // 历史题去重
        request.customCategories(),  // 自定义分类
        request.jdText()             // JD 文本
    );

    // 写 Redis：服务运行中优先从 Redis 读会话，速度快
    sessionCache.saveSession(
        sessionId,
        request.resumeText() != null ? request.resumeText() : "",
        request.resumeId(),
        questions,
        0,
        SessionStatus.CREATED
    );

    // 写 DB：Redis 过期或服务重启后，可以从数据库恢复会话
    persistenceService.saveSession(
        sessionId,
        request.resumeId(),
        questions.size(),
        questions,
        request.llmProvider(),
        skillId,
        difficulty
    );

    // 返回给前端：前端拿到 sessionId 后才能继续取题、提交答案
    return new InterviewSessionDTO(
        sessionId,
        request.resumeText() != null ? request.resumeText() : "",
        questions.size(),
        0,
        questions,
        SessionStatus.CREATED
    );
}
```

DB 保存会话：

```java
@Transactional(rollbackFor = Exception.class)
public InterviewSessionEntity saveSession(String sessionId,
                                          Long resumeId,
                                          int totalQuestions,
                                          List<InterviewQuestionDTO> questions,
                                          String llmProvider,
                                          String skillId,
                                          String difficulty) {
    InterviewSessionEntity session = new InterviewSessionEntity();

    // 对应 interview_sessions.session_id
    session.setSessionId(sessionId);

    // 对应 interview_sessions.total_questions
    session.setTotalQuestions(totalQuestions);

    // 新会话从第 0 题开始
    session.setCurrentQuestionIndex(0);

    // 新会话状态
    session.setStatus(InterviewSessionEntity.SessionStatus.CREATED);

    // questions 是 Java List，数据库 TEXT 字段只能存字符串，所以转成 JSON
    session.setQuestionsJson(objectMapper.writeValueAsString(questions));

    // 保存模型提供商，后续评估报告时继续用同一 provider
    session.setLlmProvider(llmProvider != null ? llmProvider : "default");

    // 保存面试方向和难度，用于历史题查询和展示
    session.setSkillId(skillId != null ? skillId : InterviewDefaults.SKILL_ID);
    session.setDifficulty(difficulty != null ? difficulty : InterviewDefaults.DIFFICULTY);

    // 如果有关联简历，就把 ResumeEntity 设置进去
    // JPA 会根据 @JoinColumn(name = "resume_id") 写入外键
    if (resumeId != null) {
        Optional<ResumeEntity> resumeOpt = resumeRepository.findById(resumeId);
        resumeOpt.ifPresent(session::setResume);
    }

    // save 后写入 interview_sessions
    return sessionRepository.save(session);
}
```

```text

interview_sessions.llm_provider
interview_sessions.skill_id
interview_sessions.difficulty
interview_sessions.created_at
```

## 4. 获取当前问题 GET /api/interview/sessions/{sessionId}/question

Controller：

```java
@GetMapping("/api/interview/sessions/{sessionId}/question")
public Result<Map<String, Object>> getCurrentQuestion(@PathVariable String sessionId) {
    return Result.success(sessionService.getCurrentQuestionResponse(sessionId));
}
```

Service 第一层：决定返回格式。

```java
public Map<String, Object> getCurrentQuestionResponse(String sessionId) {
    // 真正取题逻辑在 getCurrentQuestion
    InterviewQuestionDTO question = getCurrentQuestion(sessionId);

    // question == null 表示 currentIndex 已经超过题目数量
    if (question == null) {
        return Map.of(
            "completed", true,
            "message", "所有问题已回答完毕"
        );
    }

    // 没答完就返回当前题
    return Map.of(
        "completed", false,
        "question", question
    );
}
```

Service 第二层：真正从缓存/数据库取题。

```java
public InterviewQuestionDTO getCurrentQuestion(String sessionId) {
    // 先从 Redis 拿 CachedSession；Redis 没有就从 DB 恢复
    CachedSession session = getOrRestoreSession(sessionId);

    // CachedSession 里存的是 questionsJson
    // getQuestions 内部用 objectMapper.readValue 转回 List<InterviewQuestionDTO>
    List<InterviewQuestionDTO> questions = session.getQuestions(objectMapper);

    // currentIndex 是当前题下标
    // 如果下标已经大于等于题目数量，说明题目全部完成
    if (session.getCurrentIndex() >= questions.size()) {
        return null;
    }

    // 第一次取题时，状态从 CREATED 变成 IN_PROGRESS
    if (session.getStatus() == SessionStatus.CREATED) {
        // 先改内存对象状态
        session.setStatus(SessionStatus.IN_PROGRESS);

        // 同步改 Redis
        sessionCache.updateSessionStatus(sessionId, SessionStatus.IN_PROGRESS);

        // 同步改 DB
        persistenceService.updateSessionStatus(
            sessionId,
            InterviewSessionEntity.SessionStatus.IN_PROGRESS
        );
    }

    // 返回当前题
    return questions.get(session.getCurrentIndex());
}
```

`getOrRestoreSession` 是很多接口的共同入口：

```java
private CachedSession getOrRestoreSession(String sessionId) {
    // 1. 先查 Redis
    Optional<CachedSession> cachedOpt = sessionCache.getSession(sessionId);

    // 2. Redis 命中，刷新 TTL 后直接返回
    if (cachedOpt.isPresent()) {
        sessionCache.refreshSessionTTL(sessionId);
        return cachedOpt.get();
    }

    // 3. Redis 没命中，从 DB 恢复
    CachedSession restoredSession = restoreSessionFromDatabase(sessionId);

    // 4. DB 也没有，说明 sessionId 不存在
    if (restoredSession == null) {
        throw new BusinessException(ErrorCode.INTERVIEW_SESSION_NOT_FOUND);
    }

    return restoredSession;
}
```

从 DB 恢复会话：

```java
private CachedSession restoreSessionFromEntity(InterviewSessionEntity entity) {
    // questionsJson 是 interview_sessions 表里的 TEXT 字段
    // 这里把 JSON 转回题目 List
    List<InterviewQuestionDTO> questions = objectMapper.readValue(
        entity.getQuestionsJson(),
        new TypeReference<>() {}
    );

    // 答案不在 questionsJson 里长期维护，答案表才是持久化来源
    List<InterviewAnswerEntity> answers =
        persistenceService.findAnswersBySessionId(entity.getSessionId());

    // 把用户已经回答过的内容回填进题目 DTO
    for (InterviewAnswerEntity answer : answers) {
        int index = answer.getQuestionIndex();
        if (index >= 0 && index < questions.size()) {
            InterviewQuestionDTO question = questions.get(index);
            questions.set(index, question.withAnswer(answer.getUserAnswer()));
        }
    }

    // 恢复进 Redis，下次就不用查 DB
    sessionCache.saveSession(
        entity.getSessionId(),
        entity.getResume() != null ? entity.getResume().getResumeText() : "",
        entity.getResume() != null ? entity.getResume().getId() : null,
        questions,
        entity.getCurrentQuestionIndex(),
        convertStatus(entity.getStatus())
    );

    return sessionCache.getSession(entity.getSessionId()).orElse(null);
}
```

这就是“从 `CachedSession.questionsJson` 取题目列表”的完整来源。

## 5. 暂存答案 PUT /api/interview/sessions/{sessionId}/answers

暂存：保存答案，但不进入下一题。

Controller：

```java
@PutMapping("/api/interview/sessions/{sessionId}/answers")
public Result<Void> saveAnswer(@PathVariable String sessionId,
                               @RequestBody Map<String, Object> body) {
    Integer questionIndex = (Integer) body.get("questionIndex");
    String answer = (String) body.get("answer");

    SubmitAnswerRequest request = new SubmitAnswerRequest(sessionId, questionIndex, answer);
    sessionService.saveAnswer(request);

    return Result.success(null);
}
```

为什么 Controller 用 `Map<String,Object>`？

因为前端传的是 JSON：

```json
{
  "questionIndex": 0,
  "answer": "我的回答"
}
```

这里没有单独建一个 `SaveAnswerRequest`，而是用 Map 取字段，再复用 `SubmitAnswerRequest`。

Service：

```java
public void saveAnswer(SubmitAnswerRequest request) {
    // 先拿会话
    CachedSession session = getOrRestoreSession(request.sessionId());

    // Redis 里是 questionsJson，这里转成 List
    List<InterviewQuestionDTO> questions = session.getQuestions(objectMapper);

    // 取前端传来的题目下标
    int index = request.questionIndex();

    // 防止越界，比如题目只有 5 道，前端传 99
    if (index < 0 || index >= questions.size()) {
        throw new BusinessException(ErrorCode.INTERVIEW_QUESTION_NOT_FOUND);
    }

    // 取出当前题
    InterviewQuestionDTO question = questions.get(index);

    // record 不直接改字段，所以用 withAnswer 复制一个带答案的新对象
    InterviewQuestionDTO answeredQuestion = question.withAnswer(request.answer());

    // 替换 List 中原来的题
    questions.set(index, answeredQuestion);

    // 把更新后的 questions 重新序列化为 questionsJson 写回 Redis
    sessionCache.updateQuestions(request.sessionId(), questions);

    // 如果之前还是 CREATED，暂存也说明面试已经开始
    if (session.getStatus() == SessionStatus.CREATED) {
        sessionCache.updateSessionStatus(request.sessionId(), SessionStatus.IN_PROGRESS);
    }

    // 写 DB：答案表才是答案持久化来源
    persistenceService.saveAnswer(
        request.sessionId(),
        index,
        question.question(),
        question.category(),
        request.answer(),
        0,
        null
    );

    // 暂存不移动题号，只把会话状态改成进行中
    persistenceService.updateSessionStatus(
        request.sessionId(),
        InterviewSessionEntity.SessionStatus.IN_PROGRESS
    );
}
```

为什么暂存不更新 `currentQuestionIndex`？

因为用户还停留在当前题，只是把输入框内容保存下来。

## 6. 提交答案 POST /api/interview/sessions/{sessionId}/answers

提交：保存答案，并进入下一题。

Controller：

```java
@PostMapping("/api/interview/sessions/{sessionId}/answers")
@RateLimit(dimension = RateLimit.Dimension.GLOBAL, count = 10)
public Result<SubmitAnswerResponse> submitAnswer(
        @PathVariable String sessionId,
        @RequestBody Map<String, Object> body) {
    Integer questionIndex = (Integer) body.get("questionIndex");
    String answer = (String) body.get("answer");

    SubmitAnswerRequest request = new SubmitAnswerRequest(sessionId, questionIndex, answer);
    SubmitAnswerResponse response = sessionService.submitAnswer(request);

    return Result.success(response);
}
```

Service：

```java
public SubmitAnswerResponse submitAnswer(SubmitAnswerRequest request) {
    // 取会话
    CachedSession session = getOrRestoreSession(request.sessionId());

    // 从 questionsJson 反序列化题目列表
    List<InterviewQuestionDTO> questions = session.getQuestions(objectMapper);

    // 前端传来的当前题下标
    int index = request.questionIndex();

    // 校验题号合法性
    if (index < 0 || index >= questions.size()) {
        throw new BusinessException(ErrorCode.INTERVIEW_QUESTION_NOT_FOUND);
    }

    // 取当前题
    InterviewQuestionDTO question = questions.get(index);

    // 生成带用户答案的新题目对象
    InterviewQuestionDTO answeredQuestion = question.withAnswer(request.answer());

    // 替换当前题
    questions.set(index, answeredQuestion);

    // 提交后进入下一题
    int newIndex = index + 1;

    // 判断是否还有下一题
    boolean hasNextQuestion = newIndex < questions.size();

    // 如果有下一题，就取下一题；没有则为 null
    InterviewQuestionDTO nextQuestion = hasNextQuestion ? questions.get(newIndex) : null;

    // 有下一题：面试进行中
    // 没有下一题：面试完成
    SessionStatus newStatus = hasNextQuestion
        ? SessionStatus.IN_PROGRESS
        : SessionStatus.COMPLETED;

    // 先写 DB，保证持久化成功
    persistSubmittedAnswer(request, index, question, newIndex, newStatus);

    // 再更新 Redis 题目 JSON
    sessionCache.updateQuestions(request.sessionId(), questions);

    // 更新 Redis 当前题号
    sessionCache.updateCurrentIndex(request.sessionId(), newIndex);

    // 如果已经答完最后一题，更新 Redis 状态，并触发评估
    if (newStatus == SessionStatus.COMPLETED) {
        sessionCache.updateSessionStatus(request.sessionId(), SessionStatus.COMPLETED);
        enqueueEvaluationTask(request.sessionId());
    }

    // 返回给前端：是否还有下一题、下一题是什么、当前进度是多少
    return new SubmitAnswerResponse(
        hasNextQuestion,
        nextQuestion,
        newIndex,
        questions.size()
    );
}
```

提交答案的 DB 写入：

```java
private void persistSubmittedAnswer(SubmitAnswerRequest request,
                                    int index,
                                    InterviewQuestionDTO question,
                                    int newIndex,
                                    SessionStatus newStatus) {
    // 保存答案
    persistenceService.saveAnswer(
        request.sessionId(),
        index,
        question.question(),
        question.category(),
        request.answer(),
        0,
        null
    );

    // 提交答案会进入下一题，所以要更新 current_question_index
    persistenceService.updateCurrentQuestionIndex(request.sessionId(), newIndex);

    // 更新会话状态
    persistenceService.updateSessionStatus(
        request.sessionId(),
        newStatus == SessionStatus.COMPLETED
            ? InterviewSessionEntity.SessionStatus.COMPLETED
            : InterviewSessionEntity.SessionStatus.IN_PROGRESS
    );
}
```

答完最后一题触发评估：

```java
private void enqueueEvaluationTask(String sessionId) {
    // 标记评估任务已等待执行
    persistenceService.updateEvaluateStatus(sessionId, AsyncTaskStatus.PENDING, null);

    // 发消息到 Redis Stream，异步消费者会真正调用 LLM 评估
    evaluateStreamProducer.sendEvaluateTask(sessionId);
}
```

暂存和提交的区别：

| 操作 | 保存答案 | 更新 Redis questionsJson | 更新 currentQuestionIndex | 触发评估 |
|---|---|---|---|---|
| PUT 暂存 | 是 | 是 | 否 | 否 |
| POST 提交 | 是 | 是 | 是 | 最后一题触发 |

## 7. 答案为什么是 upsert

答案保存方法：

```java
@Transactional(rollbackFor = Exception.class)
public InterviewAnswerEntity saveAnswer(String sessionId,
                                        int questionIndex,
                                        String question,
                                        String category,
                                        String userAnswer,
                                        int score,
                                        String feedback) {
    // 先找会话，因为答案必须属于某个 InterviewSessionEntity
    Optional<InterviewSessionEntity> sessionOpt =
        sessionRepository.findBySessionId(sessionId);

    if (sessionOpt.isEmpty()) {
        throw new BusinessException(ErrorCode.INTERVIEW_SESSION_NOT_FOUND);
    }

    // 用 sessionId + questionIndex 查当前题是否已经有答案
    InterviewAnswerEntity answer = answerRepository
        .findBySession_SessionIdAndQuestionIndex(sessionId, questionIndex)
        .orElseGet(() -> {
            // 没有就创建
            InterviewAnswerEntity created = new InterviewAnswerEntity();

            // 设置所属会话，对应 interview_answers.session_id
            created.setSession(sessionOpt.get());

            // 设置题目下标，对应 interview_answers.question_index
            created.setQuestionIndex(questionIndex);

            return created;
        });

    // 以下字段无论新增还是更新都要覆盖
    answer.setQuestion(question);
    answer.setCategory(category);
    answer.setUserAnswer(userAnswer);
    answer.setScore(score);
    answer.setFeedback(feedback);

    // save：新对象就是 insert，已有对象就是 update
    return answerRepository.save(answer);
}
```

为什么能这样 upsert？

Entity 上有唯一约束：

```java
@UniqueConstraint(
    name = "uk_interview_answer_session_question",
    columnNames = {"session_id", "question_index"}
)
```

Repository 又提供了精确查询：

```java
Optional<InterviewAnswerEntity> findBySession_SessionIdAndQuestionIndex(
    String sessionId,
    Integer questionIndex
);
```

所以流程是：

```text
先查 sessionId + questionIndex
  -> 有：更新这条答案
  -> 没有：创建新答案
  -> save
```

```java
protected void processBusiness(EvaluatePayload payload) {
    String sessionId = payload.sessionId();

    // 查会话，同时 fetch resume，评估时需要简历文本
    Optional<InterviewSessionEntity> sessionOpt =
        sessionRepository.findBySessionIdWithResume(sessionId);

    if (sessionOpt.isEmpty()) {
        return;
    }

    InterviewSessionEntity session = sessionOpt.get();

    // 从主表 questionsJson 还原题目
    List<InterviewQuestionDTO> questions = objectMapper.readValue(
        session.getQuestionsJson(),
        new TypeReference<>() {}
    );

    // 从答案表查用户答案
    List<InterviewAnswerEntity> answers =
        persistenceService.findAnswersBySessionId(sessionId);

    // 回填答案
    for (InterviewAnswerEntity answer : answers) {
        int index = answer.getQuestionIndex();
        if (index >= 0 && index < questions.size()) {
            InterviewQuestionDTO question = questions.get(index);
            questions.set(index, question.withAnswer(answer.getUserAnswer()));
        }
    }

    // 用创建会话时保存的 llmProvider 获取对应模型客户端
    ChatClient chatClient =
        llmProviderRegistry.getChatClientOrDefault(session.getLlmProvider());

    // 简历文本来自关联的 resume
    String resumeText = session.getResume() != null
        ? session.getResume().getResumeText()
        : "";

    // 调评估服务
    InterviewReportDTO report =
        evaluationService.evaluateInterview(chatClient, sessionId, resumeText, questions);

    // 保存评估结果
    persistenceService.saveReport(sessionId, report);
}
```

```java
public void saveReport(String sessionId, InterviewReportDTO report) {
    InterviewSessionEntity session = sessionRepository.findBySessionId(sessionId)
        .orElseThrow(...);

    // 总体报告写 interview_sessions
    session.setOverallScore(report.overallScore());
    session.setOverallFeedback(report.overallFeedback());
    session.setStrengthsJson(objectMapper.writeValueAsString(report.strengths()));
    session.setImprovementsJson(objectMapper.writeValueAsString(report.improvements()));
    session.setReferenceAnswersJson(objectMapper.writeValueAsString(report.referenceAnswers()));
    session.setStatus(InterviewSessionEntity.SessionStatus.EVALUATED);
    session.setCompletedAt(LocalDateTime.now());
    sessionRepository.save(session);

    // 单题评分写 interview_answers
    for (InterviewReportDTO.QuestionEvaluation eval : report.questionDetails()) {
        InterviewAnswerEntity answer = answerMap.get(eval.questionIndex());
        answer.setScore(eval.score());
        answer.setFeedback(eval.feedback());
        answer.setReferenceAnswer(refAns.referenceAnswer());
        answer.setKeyPointsJson(objectMapper.writeValueAsString(refAns.keyPoints()));
    }

    answerRepository.saveAll(answersToSave);
}
```

为什么删除会话会删除答案？

因为会话 Entity 配了：

```java
@OneToMany(mappedBy = "session", cascade = CascadeType.ALL, orphanRemoval = true)

```

含义：

```text
删除 InterviewSessionEntity
  -> JPA 级联删除它下面的 InterviewAnswerEntity
```

注意：当前删除只删 DB，没有调用 `sessionCache.deleteSession(sessionId)`，Redis 可能短期残留到 TTL 过期。

## 11. 复现核心代码的顺序

按这个顺序写，才不会乱：

1. 先写 Entity：`InterviewSessionEntity`、`InterviewAnswerEntity`。
2. 再写 Repository：根据 `sessionId`、`resumeId + status`、`sessionId + questionIndex` 查询。
3. 再写 DTO：`CreateInterviewRequest`、`InterviewQuestionDTO`、`InterviewSessionDTO`、`SubmitAnswerRequest`。
4. 再写 Redis 缓存：`InterviewSessionCache`，重点是 `questionsJson` 序列化/反序列化。
5. 再写持久化服务：`InterviewPersistenceService`，重点是 `saveSession` 和 `saveAnswer upsert`。
6. 再写业务服务：`InterviewSessionService`，重点是 `createSession`、`getCurrentQuestion`、`saveAnswer`、`submitAnswer`。
7. 再写异步评估：`EvaluateStreamProducer`、`EvaluateStreamConsumer`。
8. 最后写 Controller：只暴露接口、接参数、调 Service、包 `Result`。

核心思维：

```text
Controller 不实现业务，只定位功能。
Service 实现业务链路。
Cache 保存进行中的会话快照。
Repository 保存可恢复的持久数据。
questionsJson 是题目列表的 JSON 形态。
interview_answers 是用户答案和单题评估的最终持久化来源。
```
